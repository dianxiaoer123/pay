查询接口

订单查询接口(可选接口)

用来主动查询订单是否支付成功，一般情况只用支付接口即可。如果您服务器中断了一段时间，可以用这个接口，快速获取中断时期的订单的支付状态。注意：此接口不能频繁查询,一个订单一分种只能查询一次

接口URL：
http://pay.tpayapi.com/get_order_staus_by_id
传参方式：Get
请求参数：
# 参数名 含义 类型 说明
1 uid 商户uid int(11) 必填。您的商户唯一标识，注册后在设置里获得。一个24位字符串
2 order_number 商户自定义订单号 string(128) 必填。
3 key 秘钥 string(32) 必填。按顺序拼接：order_number + uid + merkey，取MD5-32位加密后的值大写。

注意：merkey在安全上非常重要，一定不要显示在任何网页代码、网址参数中。只可以放在服务端。计算key时，先在服务端计算好，把计算出来的key传出来。严禁在客户端计算key，严禁在客户端存储merkey。

返回值(Json字符串)

参数内容：
    {
//1成功 0失败
"ret": 1,
"data": {
//订单号
	"order_number": "201802091026541115624518",
	//1待支付 2已支付 3支付失败
	"status": 2
},
"message": "查询成功",
"list": null,
"map": null
}
